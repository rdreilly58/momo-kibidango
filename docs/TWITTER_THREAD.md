## Tweet 2 (The Problem)

```
Here's the problem we're solving:

LLM inference is *serial*. Token by token. 256 tokens = 28 seconds.

Every real-world app needs fast responses:
- Developers want sub-second code completion
- Startups burn cash on GPU compute
- Researchers wait weeks for benchmarks
- End users abandon slow chat apps

momo-kibidango makes this faster.
```

---

## Tweet 3 (The Solution)

```
Speculative decoding: Run two models in parallel 🚀

1. Small model (3B) generates candidate tokens (FAST)
2. Large model (7B) validates in parallel (ACCURATE)
3. Accepted tokens output immediately
4. Repeat

Result: Same quality, 2x speed.
No fine-tuning. No retraining. No accuracy loss.
```

## Tweet 4 (Real Numbers)
